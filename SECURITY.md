# Security Policy

## Scope

This security policy applies only to:

- the public website at https://metawritingecology.org;
- this public website repository.

This policy does not represent the full MWE archive, Registry, working corpus,
or authority structure. It covers the public website surface and this
repository only, and does not imply that all MWE systems, archives, private
materials, models, or external services are within its scope.

## Reporting a security issue

Please report suspected security issues affecting the public website or this
repository by email to:

[security@metawritingecology.org](mailto:security@metawritingecology.org)

When available, please include:

- the affected URL, route, file, or commit;
- reproducible steps;
- the expected and the observed behavior;
- the security impact;
- relevant browser, platform, or request details;
- a minimal proof of concept that does not expose unnecessary sensitive data.

Please avoid including credentials, personal data, private archive materials,
or third-party secrets unless strictly necessary and authorized.

## Appropriate security reports

This contact is appropriate for security-relevant defects in the public
website or this repository, such as:

- exploitable website behavior;
- response-header or browser-security defects;
- repository code paths that expose secrets or unsafe execution;
- supply-chain or dependency issues that affect the deployed public website;
- unauthorized access, injection, request forgery, or data-exposure defects;
- security-relevant deployment or configuration behavior directly caused by
  this repository.

These examples are illustrative and bounded. This repository does not, by
itself, establish that the public website currently contains authentication,
private user accounts, or private databases.

## Out of scope

This security contact is not the channel for:

- conceptual disagreement;
- model naming or classification;
- Cross, Log, Protocol, Draft, or Registry status;
- relation confirmation;
- OSF priority or registration;
- editorial disagreement;
- requests to publish, hide, rename, or reclassify materials;
- ordinary broken links, copy edits, or content suggestions without a security
  impact;
- disputes about whether public surfaces represent the full MWE archive;
- preview-only platform behavior that is not caused by this repository;
- unsolicited marketing, scanning offers, or commercial services.

## No unsupported commitments

Receipt of a report does not create a response-time, resolution-time,
confidentiality, compensation, bounty, or disclosure commitment. Acceptance of
any individual report is not guaranteed.
